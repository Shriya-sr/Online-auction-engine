# Auction Engine

TLS-enabled real-time auction system with two separate desktop portals:

- a bidder client for participating in live auctions
- an admin client for starting, stopping, and monitoring auctions

The server keeps auction state on disk, supports username-based joining, tracks bidder reputation, and handles tie situations with an escalation round and a reputation-based resolution rule.

## Features

- Secure socket communication using TLS
- Separate bidder and admin desktop interfaces built with Tkinter
- Live auction status updates in real time
- Username reservation so each bidder joins with a unique name
- Bid validation against the base price and current highest bid
- Anti-sniping timer extension near the auction end
- Tie escalation rounds with blind bids
- Reputation tracking based on wins and valid bids
- Persistent auction state stored in `auction_state.json`

## Project Structure

- `server.py` - auction server, client handling, and admin command handling
- `auction.py` - auction rules, bidding logic, escalation, reputation, and persistence
- `ui_client.py` - bidder GUI client
- `admin_portal.py` - admin GUI client
- `generate_cert.py` - helper script to generate a self-signed certificate and key
- `auction_state.json` - saved auction state and reputation data
- `server.crt` / `server.key` - TLS certificate and private key used by the server and clients

## Requirements

- Python 3.10 or newer
- `tkinter` for the desktop interfaces
- `cryptography` for generating the self-signed certificate

On many Python installations, `tkinter` is already included. If it is missing, install the Tk support package for your Python distribution.

## Setup

1. Install the Python dependency:

   ```bash
   pip install cryptography
   ```

2. Generate TLS credentials if `server.crt` and `server.key` are missing:

   ```bash
   python generate_cert.py
   ```

3. Make sure you are in the project folder that contains `server.py`.

## Running the System

Open separate terminals for each process.

### 1. Start the auction server

```bash
python server.py
```

Default ports:

- bidder server: `localhost:5000`
- admin server: `localhost:5001`

### 2. Open the admin portal

```bash
python admin_portal.py
```

Use the admin portal to:

- connect to the admin port
- start an auction with item, duration, base price, and escalation window
- stop the auction manually
- request the current auction status

### 3. Open one or more bidder clients

```bash
python ui_client.py
```

Each bidder client can:

- connect to the auction server
- join with a unique username
- place bids
- request live status
- view reputation data for active users

## How It Works

The server starts two TLS listeners:

- one for bidders
- one for the admin portal

When an auction is started, the server stores the current item, base price, duration, and escalation window. Bidder clients receive live messages when bids are accepted, rejected, tied, or resolved.

If two or more bidders tie at the highest bid, the server enters an escalation round where tied bidders can submit blind higher bids for a short window. If the tie still cannot be broken by blind bids, the server resolves it using reputation score and then first-come-first-served among equal scores.

The auction also includes an anti-sniping mechanism: bids placed near the end of the auction can extend the timer so the auction cannot be won by a last-second interruption.

## Client Commands

Bidder client commands sent over the socket:

- `JOIN <username>` - join the auction with a unique username
- `BID <amount>` - place a bid
- `GET` - request auction status
- `REPUTATION` - request reputation data for active users
- `EXIT` - disconnect

Admin portal commands used internally by the GUI:

- `START <duration> <base_price> <escalation_window> <item>`
- `STOP`
- `STATUS`

## Notes

- The server expects `server.crt` and `server.key` in the project root.
- `auction_state.json` is updated automatically as the auction progresses.
- If you want to reset the system, stop the server and delete `auction_state.json`.
- The certificate generated by `generate_cert.py` is self-signed, so the clients intentionally disable certificate verification for local development.

## Example Workflow

1. Generate certificates if needed.
2. Start `server.py`.
3. Start `admin_portal.py` and press Connect.
4. Start an auction from the admin portal.
5. Start `ui_client.py` in one or more windows and join with different usernames.
6. Place bids and watch the live updates.

## Troubleshooting

- If the server fails to start, confirm that `server.crt` and `server.key` exist.
- If the GUI does not open, verify that your Python installation includes Tk support.
- If a client cannot connect, check that the server is already running and that the port numbers have not been changed.
